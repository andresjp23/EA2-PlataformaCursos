5. FLUJO DE AUTENTICACION (IMPLEMENTADO)
================================================================================

PASO A PASO:

1. CLIENTE hace POST a http://localhost:8080/auth/login
   └─ Cuerpo: { "email": "user@example.com", "password": "pass123" }

2. API GATEWAY (8080) recibe la solicitud
   └─ Identifica ruta: /auth/** 
   └─ Consulta Eureka por servicio: "auth-service"
   └─ Enruta a: http://localhost:8081/auth/login

3. AUTH-SERVICE (8081) recibe la solicitud en AuthController.login()
   └─ Extrae credenciales del LoginRequest

4. AuthenticationManager autentica credenciales
   └─ Delega a UserDetailsService

5. UserDetailServiceImpl.loadUserByUsername(email)
   └─ Consulta UserRepository.findByEmail(email)
   └─ Accede a BD: SELECT * FROM users WHERE email = ?

6. Spring Security valida contraseña
   └─ Compara contraseña en texto plano con hash BCrypt en BD
   └─ Si coincide: Authentication exitosa

7. JwtUtil.generateToken(username) genera JWT
   └─ Algoritmo: HS256
   └─ Claims: email, rol
   └─ Expira: En 1 hora (3600000ms)

8. AuthController devuelve respuesta
   └─ { "token": "eyJhbGc...", "role": "ROLE_STUDENT", "userId": 5 }

9. Cliente recibe token y lo guarda localmente
   └─ Lo incluye en futuras requests: Authorization: Bearer <token>

RESULTADO: Autenticación completada, usuario logueado
